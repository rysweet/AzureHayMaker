# Glossary - Azure HayMaker Terms

**Key terms used throughout documentation**

---

## Project Terms

**Azure HayMaker**: Orchestration service for generating benign Azure telemetry

**Agent**: Autonomous container that executes a scenario end-to-end

**Scenario**: Specific Azure technology deployment (e.g., "Linux VM Web Server")

**Orchestrator**: Service that manages agent selection, deployment, and cleanup

**Goal-Seeking Agent**: Agent that adapts to achieve objectives despite obstacles

---

## Architecture Terms

**Durable Functions**: Azure Functions with workflow orchestration (being replaced by VM)

**Container App**: Azure service for running containerized workloads (used for agents)

**Service Bus**: Azure messaging service for event streaming

**Key Vault**: Azure secret management service

**Cosmos DB**: Azure NoSQL database for logs and metrics

---

## Session Terms
